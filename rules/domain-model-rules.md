# Domain model rules
1. 도메인 모델은 외부 프레임워크나 라이브러리에 의존하지 않는 POJO(Plain Old Java Object)여야 합니다.
    - @Lombok, @Builder 등 일부는 필요에따라 유연하게 사용 (해당 라이브러리 의존의 경우 쉽게 교체 가능)
2. 도메인 모델은 비즈니스 로직을 포함해야 합니다. 단순한 데이터 구조체가 되어서는 안 됩니다.
2. 도메인 모델은 관련된 속성과 행위를 하나의 단위로 캡슐화해야 합니다.
3. 도메인 모델은 불변 객체(Immutable Object)를 활용하여 상태 변경을 명확히 표현해야 합니다.
   - 식별자가 필요하지 않은 경우(시간이나 값이 변경되어도 동일한 개체), 값 객체(Value Object)를 활용하여 도메인 모델을 설계해야 합니다.
4. 도메인 모델은 변경에 유연하게 대응할 수 있도록 설계되어야 합니다.
5. 도메인 모델은 명확한 경계를 가지고 있어야 하며, 다른 도메인 모델과의 상호작용을 최소화해야 합니다.
   - Aggregate Root 간 직접 참조를 피하고, 도메인 서비스나 이벤트를 통해 상호작용하도록 합니다. 
6. 도메인 모델은 저수준 모듈 (표현 계층이나 인프라스트럭처 계층)에 종속되지 않아야 합니다.
