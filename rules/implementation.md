# 어떤 식으로 구현해나갈것인가

1. 사용자 유스케이스를 정의한다
2. 유스케이스 별 구현 layer 를 나누어보고, 행위를 도출하여 비즈니스 로직을 선별한다.
3. 유스케이스 별 핵심 도메인로직을 추출하여 POJO 도메인 객체를 만들고 도메인 로직을 작성한다.
4. 도메인 로직을 테스트한다.
5. 유스케이스에 따른 서비스  로직을 작성한다.(repository 인터페이스 포함)
    1. 서비스를 테스트한다
    2. 서비스는 repository mock 객체를 사용한다.
6. 엔티티, repository 실체를 만든다.
7. controller 작성 및 인수테스트를 수행
